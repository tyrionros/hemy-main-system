# Hemy Main System - Process Documentation and Projects

## Overview

This repository serves as the central knowledge base for all processes and projects documented under the "Hemy Main System". It provides a standardized framework for defining, executing, and managing business processes and technical projects within the organization.

The goal is to ensure that all documentation is clear, consistent, and easy to follow.

## Core Processes

This section contains the foundational processes for the Hemy Main System.

-   [**Hemy Main Process**](./hemy-main-process.md): The foundational document outlining the core principles and stages of the Hemy Main System.
-   [**Hemy 360 - Access Request Process**](./hemy-360-access-request.md): The standardized procedure for requesting access to specific records in Dynamics 365 Field Service ("Hemy 360").

## Contributing

Contributions to this documentation are welcome. If you need to document a new process or project, please follow these steps:

1.  Use the `hemy-main-process.md` as a template for new processes.
2.  Create a new markdown file with a descriptive, lowercase name (e.g., `new-process-name.md`).
3.  Write the documentation, adhering to existing structures.
4.  Commit the new file and open a pull request for review.
