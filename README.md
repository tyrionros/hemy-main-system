# Hemy Main System - Process Documentation

## Overview

This repository serves as the central knowledge base for all processes documented under the "Hemy Main System". The Hemy Main System provides a standardized framework for defining, executing, and managing business processes within the organization.

The goal is to ensure that all processes are clear, consistent, and easy to follow.

## Documented Processes

Below is a list of the currently documented processes. Each file provides a comprehensive guide to its specific process, following the structure of the Hemy Main System.

-   [**Hemy Main Process**](./hemy-main-process.md): The foundational document outlining the core principles and stages of the Hemy Main System.
-   [**Hemy 360 - Access Request Process**](./hemy-360-access-request.md): The standardized procedure for requesting access to specific records in Dynamics 365 Field Service ("Hemy 360").

## Contributing

Contributions to this documentation are welcome. If you need to document a new process, please follow these steps:

1.  Use the `hemy-main-process.md` as a template for the structure.
2.  Create a new markdown file with a descriptive name (e.g., `new-process-name.md`).
3.  Write the documentation for the new process, adhering to the Hemy Main System stages.
4.  Commit the new file and open a pull request for review.
